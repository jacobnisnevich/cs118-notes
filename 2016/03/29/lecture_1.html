Transport View of Networking

  • Assuming application protocols take care of data content
  • Transport protocol’s job: delivering data between communicating ends
  • Don’t know or care bout which paths data may traverse through the network
  • Do care about:
    1. Delivering data to the right application process
    2. Delivery reliability
    3. Congestion control

Transport Layer Protocols

  • Unreliable data delivery: UDP (User Datagram Protocol)
  • Reliable data delivery: TCP (Transport Control Protocol)
    • Reliable delivery over potentially unreliable network
    • Understanding and managin network delays
    • Coping with congetstion
  • But transport protocols don’t really do the delivery

Network Layer’s view of the picture

  • Assuming higher layer protocols handle data content, reliability and congestionm
  • Network’s job: forward data from source to destination
  • Do care about: which way to forward data at each step?
  • Why is this hard? Because the Internet is huge!

Packet Switching: Multiplexing, Queuing Delay, Loss

  • Queuing and loss:
  • If arrival rate (in bits) to link exceeds transmission rate of link for a period of time:
    • Packets will queue, wait to be transmitted on link
    • Packets can be dropped (lost) if memory (buffer) fills up

Data Delivery Performance

Delay in packet-switched networks

  • 4 sources of delay at each hop
    • Nodal processing:
      • Check bit errors
      • Determine output link
    • Queuing = ### packets in queue X transmission time of each packets
    • Transmission = L / R
      • L = packet length (bits)
      • R = link bandwidth (bps)
    • Propagation = d / s
      • d = length of physical link
      • s = propagation speed in medium

Example: one hop delay

  • Details:
    • link length = 100 km
    • bandwidth = 1 Mbps
    • packet size = 1000 bits
    • All packets equal length
  • Queuing delay = Waiting time for 2 packets
  • Transmission delay = 1000 bits / 1000000 bits/sec = 0.001 sec = 1 msec
  • Propagation delay = 100000 m / 2 * 10^8 m/s = 0.0005 sec = 0.5 msec

Packet-switching: store-and-forward

  • Takes L / R seconds to transmit packet of L bits on to link of R bps
  • Entire packet must arrive at router before it can be transmitted on next link: store and forward

Network Latency

  • Time to send 1 packet from point A to point B
    • Sum of delays across each hop along the path
    • Delay_A-B = Delay_A-1 + Delay_1-2 + Delay_2-B
  • RTT = round-trip-time

Real internet delays and routes

  • What do “real” internet delay and loss look like?
  • traceroute program: provides delay measurement from source to router along end-end Internet path towards destination. For all i:
    • Sends three packets that will reach router i on path towards destination
    • Router i will return packets to sender
    • Sender times interval between transmission and reply
  • RTT =/= 2 * Oneway Delay traceroute
    • Paths are not necessarily symmetric
